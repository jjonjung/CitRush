# CitRush

전략 요소와 레이싱 플레이를 결합한 4인 멀티플레이 게임 프로젝트입니다.  
Unreal Engine 5.6 기반으로 차량 플레이, AI Enemy, HTTP 기반 LLM 서버 연동을 중심으로 설계했으며, 서버 장애 상황에서도 플레이가 지속되도록 비동기 처리와 로컬 AI Fallback 구조를 적용했습니다.

## 포트폴리오 링크

- 시연 영상: [CitRush](https://www.youtube.com/watch?v=OuRkQmQcT9I&t=1s)

## 프로젝트 개요

- 프로젝트: 전략형 레이싱 멀티플레이 게임
- 목표: LLM 서버 기반 AI + 4인 멀티플레이 최적화
- 기간: 2025.11 ~ 2026.01 (약 2개월)
- 인원: 5명
- 기술: Unreal Engine 5.6 / HTTP / Steam Server

## 기술 핵심

- HTTP 기반 LLM 요청 처리 서버 구조 설계
- 동기 방식에서 비동기 방식으로 전환하여 게임 진행 안정성 확보
- 서버 장애 대응을 위한 로컬 AI Fallback 구조 적용
- 컴포넌트 기반 아키텍처로 AI 및 시스템 구조 설계
- Chaos Vehicles 튜닝을 통한 차량 물리 및 네트워크 불일치 개선
- 자료구조 기반 리팩토링으로 AI 행동 로직 효율화

## 역할 및 기여도

- 총 기여도: 약 40%
- 구현 책임: 본인 담당 영역 100% 단독 구현

### 담당 범위

- 플레이 설계 및 구현
- AI 설계 및 구현
- 서버 연동 전체 담당

### 상세 기여

- 플레이어 차량 시스템 설계 및 차량 물리 처리 구현
- AI Enemy의 Enemy StateTree 및 서버 값 기반 행동 로직 구조 설계
- HTTP 기반 LLM 서버 연동 전체 설계 및 구현
- 서버 요청 처리 구조를 동기 방식에서 비동기 방식으로 전환
- 서버 장애 시에도 플레이가 지속되도록 로컬 AI Fallback 구조 구현

### 타 팀원 담당

- 지휘자
- 아웃게임 시스템
- 레벨디자인
- AI 에이전트

## 문제 해결

### 1. 차량 물리 + 네트워크 불일치

- 문제: 멀티플레이 환경에서 차량 움직임과 물리 반응이 일관되지 않는 문제가 발생
- 해결: Chaos Vehicles 세부 튜닝을 통해 차량 응답성과 네트워크 동기화 체감을 개선

### 2. AI 구조 비효율

- 문제: AI Enemy 행동 구조가 비효율적이어서 유지보수성과 확장성이 떨어짐
- 해결: 자료구조 기반으로 AI 행동 로직을 재정리하고 구조를 리팩토링

### 3. 서버 장애 시 게임 중단

- 문제: LLM 서버 요청 실패 시 게임 흐름이 중단될 위험 존재
- 해결: 비동기 처리와 로컬 AI Fallback 구조를 적용해 서버 장애 상황에서도 플레이 지속 가능하도록 개선

## 성과

- 성능: CPU 사용량 40% 감소
- 안정성: 서버 장애 상황에서도 플레이 지속 가능
- 구조: AI 및 시스템 구조 리팩토링 완료
- 최적화: 레벨 구조 개선을 통한 FPS 최적화
- AI Enemy: LLM 계산 정확도 향상
- 서비스: 멀티플레이 실환경 검증 강화
