## 에디터에서 설정

### 방법 1: Project Settings

1. **언리얼 에디터** 열기

2. **Edit** → **Project Settings** 클릭

3. **Engine** 섹션에서 **AI Subsystem** 검색
   - 또는 **Project** → **Plugins** → 프로젝트 플러그인

4. **Base AI Subsystem** 찾기

5. **AI | Config** 카테고리에서 설정 변경:
   ```
   Server URL: http://127.0.0.1
   Server Port: 8000
   Request Timeout: 10.0
   Reconnect Interval: 10.0
   Max Reconnect Attempts: 5
   ```

6. **저장** (Ctrl+S)

7. **에디터 재시작** (설정 적용을 위해)

### 방법 2: DefaultEngine.ini 직접 수정

**파일**: `Config/DefaultEngine.ini`

```ini
[/Script/UE_CITRUSH.BaseAISubsystem]
ServerURL=http://127.0.0.1
ServerPort=8000
```

**주의**: `DefaultGame.ini`가 우선 적용됩니다.

## Blueprint에서 동적 변경

### 1. 설정 변경 Blueprint

**새 Blueprint 생성**:
- **Content Browser** → 우클릭 → **Blueprint Class** → **Actor** or **Widget**
- 이름: `BP_ServerSettings`

**Blueprint Graph**:

```
Event BeginPlay
  |
  ├─> Get Game Instance
  |     |
  |     └─> Get Subsystem (BaseAISubsystem)
  |           |
  |           └─> Set Server Settings
  |                 ├─ New Server URL: http://127.0.0.1
  |                 └─ New Server Port: 8000
  |
  └─> Print String ("Server settings changed to local")
```

### 2. Blueprint 노드 설명

#### Get Subsystem (BaseAISubsystem)
```
입력: Game Instance
출력: Base AI Subsystem 레퍼런스
```

#### Set Server Settings
```
입력:
  - Target: Base AI Subsystem
  - New Server URL: String
  - New Server Port: Integer

기능:
  - 기존 연결 해제
  - 새로운 URL/Port 설정
  - Config 파일에 저장
  - 재연결 시도
```

#### Reset Server Settings
```
입력: Target (Base AI Subsystem)

기능:
  - Config 파일 기본값으로 리셋
  - 재연결
```

#### Get Server Settings
```
입력: Target (Base AI Subsystem)
출력:
  - Out Server URL: String
  - Out Server Port: Integer
```

### 3. Widget Blueprint 예시

**설정 UI Widget 생성**:

## C++에서 사용

### 1. Subsystem 가져오기

```cpp
void AMyActor::BeginPlay()
{
    Super::BeginPlay();

    // Game Instance에서 Subsystem 가져오기
    UGameInstance* GameInstance = GetGameInstance();
    if (!GameInstance)
    {
        return;
    }

    UBaseAISubsystem* AISubsystem = GameInstance->GetSubsystem<UBaseAISubsystem>();
    if (!AISubsystem)
    {
        UE_LOG(LogTemp, Error, TEXT("Failed to get BaseAISubsystem"));
        return;
    }

    // 현재 설정 확인
    FString CurrentURL;
    int32 CurrentPort;
    AISubsystem->GetServerSettings(CurrentURL, CurrentPort);
    UE_LOG(LogTemp, Log, TEXT("Current Server: %s:%d"), *CurrentURL, CurrentPort);
}
```

### 2. 런타임에 설정 변경

```cpp
void AMyActor::ChangeServerToLocal()
{
    UGameInstance* GameInstance = GetGameInstance();
    UBaseAISubsystem* AISubsystem = GameInstance->GetSubsystem<UBaseAISubsystem>();

    if (AISubsystem)
    {
        AISubsystem->SetServerSettings(TEXT("http://127.0.0.1"), 8000);
        UE_LOG(LogTemp, Log, TEXT("Server changed to local"));
    }
}

void AMyActor::ChangeServerToProduction()
{
    UGameInstance* GameInstance = GetGameInstance();
    UBaseAISubsystem* AISubsystem = GameInstance->GetSubsystem<UBaseAISubsystem>();

    if (AISubsystem)
    {
        AISubsystem->SetServerSettings(TEXT("http://34.64.141.47"), 8000);
        UE_LOG(LogTemp, Log, TEXT("Server changed to production"));
    }
}
```

### 3. Config 파일 기본값으로 리셋

```cpp
void AMyActor::ResetServerSettings()
{
    UGameInstance* GameInstance = GetGameInstance();
    UBaseAISubsystem* AISubsystem = GameInstance->GetSubsystem<UBaseAISubsystem>();

    if (AISubsystem)
    {
        AISubsystem->ResetServerSettings();
        UE_LOG(LogTemp, Log, TEXT("Server settings reset to config defaults"));

```

### 4. Widget에서 사용

```cpp
// SettingsWidget.h
UCLASS()
class USettingsWidget : public UUserWidget
{
    GENERATED_BODY()

public:
    UPROPERTY(meta = (BindWidget))
    class UEditableTextBox* ServerURLTextBox;

    UPROPERTY(meta = (BindWidget))
    class UEditableTextBox* ServerPortTextBox;

    UPROPERTY(meta = (BindWidget))
    class UButton* ApplyButton;

    UPROPERTY(meta = (BindWidget))
    class UButton* ResetButton;

protected:
    virtual void NativeConstruct() override;

    UFUNCTION()
    void OnApplyButtonClicked();

    UFUNCTION()
    void OnResetButtonClicked();

private:
    UPROPERTY()
    class UBaseAISubsystem* AISubsystem;
};

// SettingsWidget.cpp
void USettingsWidget::NativeConstruct()
{
    Super::NativeConstruct();

    // Subsystem 가져오기
    UGameInstance* GameInstance = GetGameInstance();
    AISubsystem = GameInstance->GetSubsystem<UBaseAISubsystem>();

    // 현재 설정 로드
    if (AISubsystem)
    {
        FString CurrentURL;
        int32 CurrentPort;
        AISubsystem->GetServerSettings(CurrentURL, CurrentPort);

        ServerURLTextBox->SetText(FText::FromString(CurrentURL));
        ServerPortTextBox->SetText(FText::AsNumber(CurrentPort));
    }

    // 버튼 이벤트 바인딩
    if (ApplyButton)
    {
        ApplyButton->OnClicked.AddDynamic(this, &USettingsWidget::OnApplyButtonClicked);
    }

    if (ResetButton)
    {
        ResetButton->OnClicked.AddDynamic(this, &USettingsWidget::OnResetButtonClicked);
    }
}

void USettingsWidget::OnApplyButtonClicked()
{
    if (!AISubsystem)
    {
        return;
    }

    FString NewURL = ServerURLTextBox->GetText().ToString();
    int32 NewPort = FCString::Atoi(*ServerPortTextBox->GetText().ToString());

    // 설정 적용
    AISubsystem->SetServerSettings(NewURL, NewPort);

    UE_LOG(LogTemp, Log, TEXT("Server settings applied: %s:%d"), *NewURL, NewPort);
}

void USettingsWidget::OnResetButtonClicked()
{
    if (!AISubsystem)
    {
        return;
    }

    // 기본값으로 리셋
    AISubsystem->ResetServerSettings();

    // UI 업데이트
    FString CurrentURL;
    int32 CurrentPort;
    AISubsystem->GetServerSettings(CurrentURL, CurrentPort);

    ServerURLTextBox->SetText(FText::FromString(CurrentURL));
    ServerPortTextBox->SetText(FText::AsNumber(CurrentPort));

    UE_LOG(LogTemp, Log, TEXT("Server settings reset to: %s:%d"), *CurrentURL, CurrentPort);
}
```

## 테스트 방법

### 1. Config 파일 테스트

**단계**:
1. `Config/DefaultGame.ini` 파일 열기
2. `ServerURL`을 `http://ip`로 변경
3. 에디터 재시작
4. Play (PIE) 실행
5. Output Log에서 확인:
   ```
   [BaseAISubsystem] Initializing - Server: http://127.0.0.1:8000
   ```

### 2. Blueprint 동적 변경 테스트

**단계**:
1. `BP_ServerSettings` Blueprint 생성 (위 예시 참고)
2. Level에 배치
3. Play (PIE) 실행
4. Output Log 확인:
   ```
   [BaseAISubsystem] SetServerSettings - Old: http://34.64.141.47:8000, New: http://127.0.0.1:8000
   [BaseAISubsystem] Server settings changed - New: http://127.0.0.1:8000
   ```

### 3. Widget UI 테스트

**단계**:
1. `WBP_ServerSettings` Widget 생성
2. PlayerController에서 Widget 표시
3. URL/Port 입력 후 **Apply** 클릭
4. Output Log 확인

### 4. 실제 서버 연결 테스트

**로컬 서버 테스트**:
```
1. Config에서 ServerURL=http://127.0.0.1
2. 로컬에서 AI 서버 실행 (http://127.0.0.1:8000)
3. Play (PIE)
4. Output Log 확인:
   [BaseAISubsystem] Connected to AI Server 연결 성공
```

**프로덕션 서버 테스트**:
```
1. Config에서 ServerURL=http://34.64.141.47
2. Play (PIE)
3. Output Log 확인
```

## 주의사항

### 1. Config 파일 우선순위

```
Blueprint SetServerSettings() > Config 파일 > 코드 기본값
```

### 2. 에디터 재시작

Config 파일 변경 후 **반드시 에디터 재시작** 필요

### 3. SaveConfig()

`SetServerSettings()` 함수는 자동으로 `SaveConfig()`를 호출하여 Config 파일에 저장합니다.

다음 실행 시에도 마지막 설정이 유지됩니다.

### 4. 유효성 검증

`SetServerSettings()`는 다음을 검증합니다:
- URL이 비어있지 않음
- Port가 1~65535 범위

### 5. 재연결

`SetServerSettings()` 또는 `ResetServerSettings()` 호출 시 자동으로:
1. 기존 연결 해제
2. 새 설정 적용
3. 재연결 시도

## 트러블슈팅

### Config 파일이 적용 안 됨

**증상**: Config 수정했는데 기본값 사용

**해결**:
1. 에디터 완전 종료 후 재시작
2. `Saved/Config` 폴더 삭제 후 재실행
3. Project Settings에서 직접 수정

### Blueprint 노드가 안 보임

**증상**: `Set Server Settings` 노드를 찾을 수 없음

**해결**:
1. 프로젝트 재컴파일 (Ctrl+Alt+F11)
2. Blueprint 리프레시 (File → Refresh All Nodes)
3. `UFUNCTION(BlueprintCallable)` 지정자 확인

### 설정이 저장 안 됨

**증상**: `SetServerSettings()` 호출 후 재시작하면 원래 값으로 돌아감

**해결**:
- `SaveConfig()` 호출 확인 (이미 구현되어 있음)
- Config 파일 쓰기 권한 확인
- `Config/` 폴더 읽기 전용 속성 해제
